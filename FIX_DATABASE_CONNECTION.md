# Быстрое исправление подключения к базе данных

## Вариант 1: Connection Pooling (РЕКОМЕНДУЕТСЯ - попробуйте сначала)

1. Откройте **Supabase Dashboard** → **Settings** → **Database**
2. Найдите секцию **Connection Pooling**
3. Скопируйте строку подключения (порт **6543**, не 5432)
4. Обновите в `.env`:
   ```
   DATABASE_URL=postgresql://postgres.[PROJECT-REF]:[PASSWORD]@aws-0-[REGION].pooler.supabase.com:6543/postgres?pgbouncer=true
   ```
5. Перезапустите сервер: `npm run dev`

**Почему это работает:**
- Pooler использует другой домен (pooler.supabase.com вместо db.*)
- Более надежное подключение
- Готово для serverless

## Вариант 2: Смена DNS (если Вариант 1 не помог)

Если проблема в DNS провайдера:

1. Откройте **Настройки сети Windows**
2. Измените DNS на:
   - **1.1.1.1** (Cloudflare) или
   - **8.8.8.8** (Google)
3. Перезапустите компьютер
4. Попробуйте снова

## Вариант 3: Переход на Supabase API (если ничего не помогло)

Если оба варианта не работают - переходим на Supabase API (как рекомендует GPT).

Это потребует рефакторинга, но решит проблему навсегда.
